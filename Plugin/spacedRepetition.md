# Spaced Repetition

## 0. 核心概念：两套系统，别混在一起

Spaced Repetition 里有两条复习体系：

1. **Flashcards（卡片复习）**

- 单位：一张张卡
- 你写 `问题::答案` / 挖空等，插件按卡片排程

1. **Notes Review（整篇笔记复习）**

- 单位：整篇笔记
- 你给笔记加 `review` 标签，插件按“笔记”排程（右侧队列显示“今天/明天/之后”）

> **结论：不需要必须两个都加。**
>  卡片用 flashcards，整篇通读回顾用 review。想两者都要也可以，但工作量会更大。

------

## 1. Flashcards：如何创建卡片（你已掌握，给你完整整理）

### 1.1 必须先让插件“识别到卡片属于某个 Deck”

最常用方案：**用标签作为 Deck**

- 在插件设置：卡片标签（你现在是 `#flashcards`）
- 在笔记属性 tags 里加 `flashcards` 就会被识别为一个 deck

> 推荐用层级标签做子牌组：
>  `#flashcards/链表/排序链表`、`#flashcards/二分/模板` 等

### 1.2 单行卡

- 单向卡：
   `问题::答案`
- 双向卡（会生成两张互为正反）：
   `A:::B`

### 1.3 多行卡（需要分隔符）

- 多行单向卡：

  ```
  问题第一行
  问题第二行
  ?
  答案第一行
  答案第二行
  ```

- 多行双向卡：用 `??`（取决于你设置）

### 1.4 Cloze 挖空卡

- 默认高亮挖空：
   `这是 ==答案==`
- 你插件设置里还可以把 `==高亮==`、`**加粗**`、`{{}}` 等当作挖空来源（按你设置为准）

## 2. Flashcards：复习模式（最重要！你之前卡住就是因为这个）

Flashcards 有两种“复习入口”，它们行为完全不同：

### 2.1 Review（到期复习：只出 New + Due）

- 特点：**只复习“新卡 + 到期卡”**
- 所以当你当天做完后：`到期=0，新卡=0` → 再点 Review 会看起来“没反应”

你在命令面板里对应的命令是：

- **Spaced Repetition：复习此笔记中的卡片**
- **Spaced Repetition：复习所有笔记中的卡片**

### 2.2 Cram / 集中复习（主动回顾：不管到期与否）

- 特点：**把“全部卡”拿出来刷**（临考冲刺/当天二刷用）
- 你中文界面里叫：**“集中复习”**

对应命令是：

- ✅ **Spaced Repetition：集中复习此笔记中的卡片**（你已验证可用，绕过卡组窗口）
- **Spaced Repetition：选择要集中复习的卡组**（会弹出“卡组列表窗口”）

> 你这次的最佳实践：
>  **如果卡组列表窗口点不动，直接用“集中复习此笔记中的卡片”绕过它。**
>  这是最稳定、最省事的主动复习方式。

## 3. 评分按钮（简单/记得/较难）什么时候才会生效？

你在 `Ctrl+P` 看到的：

- 标记为“简单”
- 标记为“记得”
- 标记为“较难”

这些是**对当前正在复习的卡/笔记进行评分**的。
 所以正确顺序是：

1. 先进入复习会话（Review 或 集中复习）
2. 出现卡片正面/答案按钮后
3. 再按简单/记得/较难（或用快捷键）

> 在普通阅读模式直接点“标记为简单/较难”，很多时候会感觉“没变化”，因为没有 active 的复习对象。

------

## 4. Notes Review（#review）：为什么只它会在右侧队列显示？

右侧那个“今天/明天/之后”的队列是 **笔记复习队列（Note Review Queue）**，只服务于整篇笔记复习。

### 4.1 如何启用 Notes Review

- 给笔记加标签：`review`
- 插件设置里确保“Notes review tags / Tags to review”包含 `review`
- 用命令打开队列：
  - **Spaced Repetition：在侧边栏中打开笔记复习队列**
  - 或：**打开一个笔记开始复习**

### 4.2 Notes Review 的打分逻辑

进入笔记复习流程后，对整篇笔记打：

- 简单/记得/较难
   插件会把排程写进笔记属性（常见字段：`sr-due / sr-interval / sr-ease`）

------

## 5. SR 注释是什么？能不能删？（你问到的重点）

Flashcards 的排程数据会写在卡片旁边，例如：
 `<!--SR:!2025-12-16,4,270-->`

### 5.1 它的意义

这段注释记录：

- 下次到期日（due）
- 间隔（interval）
- 难度/易度（ease）等

### 5.2 删除注释会怎样？

- ✅ 会让这张卡变回 **新卡（New）**
- ❌ 也会清空它的学习历史（相当于“从头学”）

### 5.3 正确使用建议

- **想二刷回顾：用“集中复习（Cram）”，不要删注释**
- **想重置/学崩了从零开始：可以删注释**（或用插件自带 Reset，如果你的版本有）

------

## 6. 你遇到的“卡组列表窗口点不动”问题：结论与解决方案

现象：卡组窗口里 `flashcards` 行可以 hover 高亮，但单击/双击/回车无效。

### 6.1 最稳绕过方案（你已验证）

- ✅ `Ctrl+P` → **集中复习此笔记中的卡片**
   直接开刷，不依赖卡组列表窗口。

### 6.2 想彻底修好（可选）

按优先级排查：

1. 更新 Spaced Repetition 插件 + 更新 Obsidian
2. 切换到默认主题 + 关闭所有 CSS snippets + Reload app
3. 临时关闭其他第三方插件（只保留 SR）排查冲突
4. `Ctrl+Shift+I` 查看 Console 是否有 SR 报错

> 但实际使用上：你已经有“绕过方案”，可以把修复放在以后不着急。

------

## 7. 推荐工作流（给你一套最省事的复习习惯）

### 日常（按排程复习）

- `Ctrl+P` → **复习所有笔记中的卡片**（Review：只出到期+新卡）
- `Ctrl+P` → **打开一个笔记开始复习**（Notes Review）

### 当天想再回顾/临考冲刺

- 打开目标笔记 → `Ctrl+P` → ✅ **集中复习此笔记中的卡片**（Cram）
- 想刷某个 deck：如果卡组窗口可用再用“选择要集中复习的卡组”，不可用就用“笔记内集中复习”替代

### 组织结构建议

- 每道题/每个知识点一篇笔记
- 需要背的点写成卡（flashcards）
- 需要隔几天通读的笔记加 review

------

## 8. 最常见误区速查

- “我今天复习完了，再点复习没反应”
   → 因为 Review 只出 Due/New，今天已经 0 了；用 **集中复习** 二刷。
- “为什么 flashcards 不在右侧队列显示？”
   → 右侧队列是 Note Review Queue，只显示 `#review` 的笔记。
- “我按简单/较难没变化”
   → 必须先进入复习会话再评分。
- “删 SR 注释就能主动复习？”
   → 那是 **重置为新卡**，不是正常回顾方式。
